# 김휘성 API 문제 정리

> 작성일: 2025-12-13

## 🔍 문제 상황

김휘성님이 "API는 다 확인했고"라고 했지만, 브라우저에서 504 타임아웃 발생.

## ✅ 확인된 사실

1. **API 자체는 정상 작동**:
   - 직접 호출 시: **200 OK** (약 2.25초) ✅
   - 데이터 정상 반환 ✅

2. **브라우저에서만 타임아웃**:
   - 브라우저 호출 시: **504 Gateway Timeout** (약 30초)
   - 실제 타임아웃: 약 30초

3. **타임아웃 설정**:
   - Lambda 함수 타임아웃: **28초** (이미 조정 완료) ✅
   - API Gateway Integration 타임아웃: **29초** ❌
   - 프론트엔드 타임아웃: **60초** ✅

## 💡 문제 원인

**API Gateway Integration 타임아웃 (29초)이 Lambda 함수 타임아웃 (28초)보다 길지만, 실제로는 29초에 타임아웃 발생**

- Lambda 함수는 28초까지 실행 가능
- 하지만 API Gateway가 29초에 타임아웃
- 실제로 약 30초에 타임아웃 발생 (29초 + 지연)

## 🎯 해결 방법

**서성덕님께 API Gateway Integration 타임아웃을 30초 이상으로 조정 요청**

## 📋 김휘성님 할 일

**없음. 이미 완료됨.**

- ✅ Lambda 함수 타임아웃 28초로 조정 완료
- ✅ API 자체는 정상 작동 확인 완료

## 📋 서성덕님 할 일

**API Gateway Integration 타임아웃을 30초 이상으로 조정**
